# 65C02

## 1.3  65C02 (CMOS)

New stable instructions vs NMOS:

```
BRA  $80  — relative branch (2 bytes, 1 extra cycle over Bcc)
STZ  $64+ — store zero to abs/abs,X/zp/zp,X
TRB  $14+ — test+reset bits; TSB $04+ — test+set bits
BIT zp    — new zero-page form
BIT abs,X — new absolute form
BIT #imm  — immediate query (N/MSB, Z, V unchanged)
PHY/PLY $5A/$7A — push/pull Y
PHX/PLX $DA/$FA — push/pull X
WAI $CB — wait for IRQ
STP $DB — stop clock
```

Removed: all 6502 undocumented opcodes have NOP or reassigned replacements.

Decimal mode: N flag = bit7 of result (same as binary); V flag likewise. ADC/SBC take +1 cycle in D mode. D flag auto-clears on reset/IRQ on 65C02 unlike 6502.

### CPU family detection without undocumented opcodes

The NMOS 6502 and CMOS 65C02 differ in decimal-mode flag behavior. In BCD,
`$99 + $01` produces zero; 65C02 sets Z from the corrected decimal result,
while NMOS 6502 leaves Z as if the binary add happened.

```asm
; out: A=0 NMOS 6502, A=1 65C02, A=2 65C816
; assembled with 65C816 mnemonics enabled; no undocumented 6502 opcodes
detect_cpu
        lda   #$99
        clc
        sed
        adc   #$01
        cld
        beq   @cmos
        lda   #$00
        rts

@cmos   rep   #$02              ; 65C816 clears Z; 65C02 treats as inert/NOP-like
        bne   @c816
        lda   #$01
        rts
@c816   lda   #$02
        rts
```

Keep `CLD` in the routine even if the caller promises binary mode. Decimal mode
leaking into Atari OS, interrupt, or math code is a high-value bug.
